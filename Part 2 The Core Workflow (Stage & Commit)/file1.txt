1. git add -p	"Patch" mode.  -> Allows you to stage parts of a file interactively.

when you update any file then this commond will show in terminal what changed happen and new code also . 

after this commond - "git add -p"  you will get this 

| Key | Meaning                   |
| --- | ------------------------- |
| `y` | Stage this change         |
| `n` | Skip this change          |
| `q` | Quit                      |
| `a` | Stage all remaining       |
| `d` | Skip all remaining        |
| `s` | Split into smaller chunks |
| `e` | Edit manually             |
| `?` | Help                      |












2. git rm <file>	Deletes a file from both the working directory and the index.

index - It’s a temporary area where Git stores changes before committing.
👉 The index (staging area) is:
A place where you prepare changes before saving them permanently using commit.

This commond is used when you run these commonds - 
          git add . 
          git commit -m "__"
you will see the list of files changed ... 

Now if you want any file not commited in the main , then you can run -           git rm "__"  
=> it will remove file from the local and the index also...











3.  git rm --cached <file>	Removes a file from Git tracking but keeps it on your local disk.
